2. tirth8205/code-review-graph:
   star: 24740  语言/技术: Python, Tree-sitter, MCP (Model Context Protocol), Tree-sitter语言包, GitHub Actions
   项目简介: 一个面向AI代码审查的结构化图谱工具，通过Tree-sitter解析代码AST，构建函数、类、导入等节点的图，并利用MCP协议为AI助手提供精准上下文，避免大模型重复读取整个代码库。支持增量更新、爆炸半径分析、CI风险评分，兼容30+语言和主流AI编程工具（如Cursor、Claude Code等）。一句话：让AI审查只读该读的代码，节省80%以上的token消耗。
   亮点: 增量更新 <2秒：仅重新解析变更文件，依赖SHA-256哈希，2900文件项目只需2秒。; 爆炸半径分析：自动追踪变更影响的所有调用者、依赖和测试，AI只读取相关文件。; CI集成GitHub Action：PR评论中按风险评分函数，可设置失败门槛，无需上传代码到外部服务。
   详细报告: code-review-graph是一个专为AI代码审查设计的开源工具，核心思想是通过构建代码的结构化知识图谱，让AI助手在审查代码时只读取最相关的内容，而不是扫描整个代码库。它使用Tree-sitter将代码解析为抽象语法树（AST），然后转换为图结构，其中节点代表函数、类、导入等，边代表调用、继承、测试覆盖等关系。

主要功能包括：
1. **图构建与查询**：通过`code-review-graph build`命令解析整个代码库，生成图；后续AI可调用MCP工具查询特定符号或变更的影响范围。
2. **增量更新**：使用Git/SVN钩子或watch模式，文件保存或提交时自动更新图，仅重新解析变化部分，速度极快（<2秒）。
3. **爆炸半径分析**：当文件变更时，图跟踪所有潜在影响的函数、类和测试文件，AI只读取这些文件，而非整个项目。例如在monorepo中，27700+文件被排除，仅读取约15个文件。
4. **语言支持广泛**：通过Tree-sitter支持Python、JavaScript/TypeScript/TSX、Go、Rust、Java、C/C++、C#、Ruby等30+语言，包括Jupyter笔记本。用户还可通过自定义`languages.toml`添加新语言，无需修改源码。
5. **CI集成**：作为GitHub Action运行，在PR中发布粘性评论，显示风险评分的函数、受影响的执行流和测试缺口。可配置`fail-on-risk`作为合并门禁。
6. **Token节省显著**：基准测试显示，中位数每问题token减少约82倍（范围38x-528x）。
7. **本地优先**：所有解析和查询在本地或CI运行器上完成，不发送代码到外部服务，可选开启云嵌入。

技术实现上，它依赖Python 3.10+，推荐使用uv工具。安装后通过`code-review-graph install`自动检测并配置Cursor、Claude Code、Copilot等主流AI工具。项目还提供了详细的基准测试和文档，包括FAQ、故障排除等。

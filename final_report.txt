   项目简介: SkillOpt 是一个用于自我进化智能体技能的训练框架，灵感来源于深度学习训练流程。它把技能文档当作可训练的状态，通过优化器模型对技能文档进行增、删、改的编辑操作，并采用验证门控机制确保每次编辑都能提升性能。训练后的技能是一个紧凑的 markdown 文件（约300-2000 token），部署时零额外推理开销。该项目支持多种大模型后端（OpenAI、Azure、Claude、Qwen 等）和多种执行环境（直接对话、Codex CLI、Claude Code CLI），并在多个基准测试中取得了最佳或并列最佳的结果。
   亮点: 将技能文档视为可训练状态，像训练神经网络一样训练技能（epoch、batchsize、学习率、验证门控）; 零推理时模型调用，部署后不增加任何延迟; 在6个基准、7个模型、3种执行器上共52个评估单元中全部最佳或并列最佳
   详细报告: SkillOpt 是微软推出的一个开源框架，核心思想是让智能体的技能（skill，即一段提示词或行为文档）能够像神经网络权重一样通过训练来自动优化。传统上，智能体技能靠手工编写或一次生成，缺乏可靠的改进机制。SkillOpt 把技能文档作为可训练的状态，使用一个独立的优化器模型（比如一个较强的LLM）对技能文档进行编辑操作（增加、删除、替换），并通过一个 held-out 验证集来评估每次编辑的效果，只有提升分数的编辑才会被采纳。训练过程引入了 epoch、mini-batchsize、学习率预算（文本形式）、拒绝编辑缓冲等机制，使技能训练稳定可靠。训练产物是一个紧凑的 best_skill.md 文件（300-2000 token），部署时直接用于原来的目标模型，不增加任何额外模型调用。该项目支持多种后端：OpenAI、Azure OpenAI、Claude、Qwen、MiniMax 等，以及多种执行环境：直接对话、Codex CLI、Claude Code CLI、Cursor 等。它还提供了 WebUI 监控仪表盘（基于 Gradio）。在论文中，SkillOpt 在 6 个基准（SearchQA 等）、7 个目标模型（包括 GPT-5.5）、3 种执行器上共 52 个评估单元全部达到最佳或并列最佳，平均提升无技能基线 23.5 分（直接对话）、24.8 分（Codex 循环）、19.1 分（Claude Code 循环）。此外，优化的技能可以在不同模型规模、不同执行器之间转移。该项目还推出了 v0.2.0 版本，包含 SkillOpt-Sleep 夜间离线自进化引擎。技术栈涉及 Python、多种 LLM API、CLI 工具、Gradio、YAML 配置等。文档详细，包括安装、数据准备、训练/评估命令、配置、新后端/新基准扩展指南等。

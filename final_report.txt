   项目简介: Headroom 是一个专为 AI Agent 设计的「上下文压缩层」，核心功能是在数据到达大模型（LLM）之前，对工具输出、日志、RAG 分块、文件及对话历史进行 60%-95% 的 Token 压缩，同时保证答案质量不下降。它支持 Python/TypeScript 库、零代码改动代理（Proxy）、MCP 服务器、多种 AI 编程助手（Claude Code, Cursor, Codex 等）以及跨 Agent 共享记忆。其技术亮点包括 6 种压缩算法（JSON SmartCrusher、AST CodeCompressor、HuggingFace Kompress-base 模型等）、可逆压缩（CCR 按需还原原文）、输出 Token 削减（减少模型废话）以及基于失败会话自动学习的 headroom learn 功能。简而言之，这是一个让 AI Agent 跑得更省、更快、更聪明的“中间件神器”。
   亮点: 极致 Token 压缩：实测在代码搜索、SRE 调试等场景实现 73%-92% 的 Token 节省，直接降低 API 调用成本; 可逆压缩（CCR）：原始数据本地缓存，LLM 需要时可随时检索还原，兼顾压缩与数据完整性; 输出 Token 削减：不仅压缩输入，还通过 Verbosity Steering 和 Effort Routing 减少模型输出中的废话，进一步省钱; 跨 Agent 记忆共享：支持 Claude、Codex、Gemini 等多 Agent 间的去重共享存储，提升多 Agent 协作效率; 零代码集成：通过 Proxy 模式一行命令即可接入现有项目，支持主流框架（LangChain, Agno, Strands 等）和 MCP 协议
   详细报告: Headroom 解决的核心痛点是：现代 AI Agent（如 Claude Code、Cursor、GitHub Copilot CLI 等）在与大模型交互时，会产生大量冗余上下文（工具输出、日志、RAG 结果、历史对话），导致 Token 成本飙升、响应延迟增加，甚至因上下文窗口限制而丢失关键信息。

该项目的技术架构分为几个核心模块：
1. **ContentRouter**：智能识别内容类型（JSON、代码、文本、图片），路由到对应的压缩器。
2. **SmartCrusher**：针对 JSON 数据（如 API 返回值、日志）进行结构化压缩，支持嵌套对象和混合类型。
3. **CodeCompressor**：基于 AST（抽象语法树）感知的代码压缩，支持 Python、JS、Go、Rust、Java、C++ 等多种语言，能智能裁剪无关代码片段。
4. **Kompress-base**：自研的 HuggingFace 模型（kompress-v2-base），专门针对 Agent 对话轨迹训练，擅长压缩自然语言文本。
5. **CacheAligner**：稳定前缀以命中 LLM 提供商的 KV Cache，减少重复计算。
6. **CCR（Compress, Cache, Retrieve）**：可逆压缩机制，原始数据本地存储，LLM 需要时通过 headroom_retrieve 工具按需还原。

部署方式极其灵活：
- **Library 模式**：`compress(messages)` 直接嵌入 Python/TypeScript 代码。
- **Proxy 模式**：`headroom proxy --port 8787` 启动本地代理，零代码改动即可拦截所有 LLM 请求。
- **Agent Wrap 模式**：`headroom wrap claude|codex|cursor` 一行命令包装现有 AI 编程助手。
- **MCP Server 模式**：提供 `headroom_compress`、`headroom_retrieve`、`headroom_stats` 工具，任何 MCP 客户端均可调用。

实测效果惊人：在代码搜索（100 结果）场景下从 17,765 Token 压缩到 1,408 Token（92% 压缩率）；SRE 故障调试从 65,694 Token 压缩到 5,118 Token；且在 GSM8K 数学基准和 TruthfulQA 事实性基准上准确率无损失甚至略有提升。

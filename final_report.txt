   详细报告: 该项目名为 code-review-graph，旨在解决 AI 代码审查工具在审查时因重新读取大量代码库而浪费 token 的问题。其核心原理是利用 Tree-sitter 解析代码库生成抽象语法树（AST），并存储为包含节点（函数、类、导入等）和边（调用、继承、测试覆盖等）的结构化图。在每次代码变更时，通过该图进行最小上下文计算，只向 AI 助手提供影响范围内的文件，避免全量扫描。

- **增量更新**：通过 SHA-256 哈希检测变化文件，仅重新解析变更部分，2900 文件的仓库可在 2 秒内完成索引。
- **爆炸半径分析**：追踪变更影响的调用者、依赖项和测试文件，生成最小影响集。
- **语义搜索**：可选集成向量嵌入（如 sentence-transformers）提升搜索准确性。
- **多平台支持**：可配置用于 Codex、Cursor、Claude Code、GitHub Copilot 等主流 AI 编码工具。
- **GitHub Action**：在 PR 时自动生成风险评分、影响函数和测试缺口，可设置 fail-on-risk 作为合并门禁。
- **自定义语言支持**：用户可通过 languages.toml 添加新语言解析规则，无需修改源码。
- **基准测试**：在 6 个真实开源仓库上测试，平均 token 减少 82 倍，平均影响精度 F1 为 0.71。

技术栈主要基于 Python，依赖 Tree-sitter 解析器和 MCP 协议。安装简单（pip install code-review-graph），并使用 code-review-graph install 命令自动配置。项目文档详尽，包含 FAQ、使用指南、故障排除等，社区活跃度高（Discord）。

   项目简介: turbovec 是一个基于 Google TurboQuant 算法的高效向量索引，用 Rust 编写并提供 Python 绑定。它能在 4GB 内存中容纳原本需要 31GB 的 1000 万文档向量（1536 维），搜索速度比 FAISS 还快 10-19%（ARM 上），并且无需训练、即插即用。支持过滤搜索、SIMD 加速、纯本地部署，非常适合隐私敏感的 RAG 场景。

   详细报告: turbovec 是一个基于 Rust 的高性能向量索引库，实现了 Google 的 TurboQuant 算法（ICLR 2026）。其核心思想是利用随机旋转使坐标分布变得可预测，然后使用 Lloyd-Max 标量量化将每个浮点坐标压缩为 2 位或 4 位整数，最后进行位打包。具体流程：1) 归一化：去除向量长度，存储为 float；2) 随机旋转：乘以同一个随机正交矩阵，使坐标独立服从 Beta 分布；3) 坐标校准（TQ+）：拟合两个标量调整坐标分布，提高低维精度；4) Lloyd-Max 量化：根据已知分布预计算最优桶边界和质心，无需数据训练；5) 位打包：将量化后的整数紧密打包到字节中。搜索时，查询向量经过同样旋转后，直接在压缩域内用 SIMD 指令打分，无需解压缩。同时支持长度重归一化（LRN）校正量化偏差，使分数无偏。项目提供 Python 绑定（通过 maturin 构建），可 pip 安装，并内置了 LangChain、LlamaIndex、Haystack、Agno 等框架的集成，可一键替换内存向量存储。性能方面，在 100K 向量、k=64 的基准测试中，TurboQuant 在 ARM 上全面超越 FAISS FastScan 10-19%，在 x86 上 4-bit 配置胜出，2-bit 配置稍落后。压缩率方面，1536 维向量从 6KB 降至 384 字节（2-bit）或 768 字节（4-bit），内存节省高达 16 倍。此外，项目支持 O(1) 删除的 IdMapIndex 和基于 ID 允许列表的高效过滤搜索，过滤逻辑集成在 SIMD 核内，避免不必要计算。

   star: 121631  语言/技术: Python, uv (Python包管理器), Markdown, GitHub API, AI编码代理集成 (Claude Code, Copilot等)
   项目简介: Spec Kit 是一个开源工具包，旨在通过规范驱动开发（Spec-Driven Development）来加速高质量软件的构建。它颠覆了传统“代码为王”的模式，让规范成为可执行的核心——开发者只需描述“做什么”和“为什么”，工具便会自动生成任务列表和技术实现计划。支持30+ AI编码代理（如Claude Code、GitHub Copilot），通过slash命令实现从编写规范、生成计划、分解任务到自动实现的完整工作流。还提供扩展、预设和捆绑包机制，允许用户按角色定制开发体验。
   亮点: 规范即代码：将需求文档转化为可自动执行的开发流程，减少“随性编码”带来的不确定性。; 与30+ AI编码代理深度集成：通过slash命令在Agent内直接调用，无需切换工具。; 高度可扩展：支持自定义扩展、预设和捆绑包，适合团队按角色（PM、开发者、安全研究员等）定制工作流。
   详细报告: Spec Kit 由GitHub官方推出，核心理念是规范驱动开发（SDD）。传统开发中，代码是最终产物，规范只是辅助文档；而在SDD中，规范本身就是可执行的蓝图。

**核心工作流**：
1. 使用 `specify init` 初始化项目，选择AI代理集成。
2. 通过 `/speckit.constitution` 设定项目原则和开发指南。
3. 用 `/speckit.specify` 描述需求（关注“做什么”和“为什么”）。
4. 用 `/speckit.plan` 制定技术实现计划（指定技术栈和架构）。
5. 用 `/speckit.tasks` 分解为可执行的任务列表（包含依赖管理和并行标记）。
6. 最终用 `/speckit.implement` 自动执行所有任务。
7. 还有可选命令如 `/speckit.clarify`（澄清未明确点）、`/speckit.analyze`（一致性分析）、`/speckit.checklist`（生成质量清单）等。

**技术构成**：
- **CLI工具**：基于Python，通过 `uv tool install` 安装，支持自升级。
- **集成层**：支持30+AI代理（包括Claude Code, Copilot Chat, Cody等），提供slash命令或技能模式。
- **可扩展性**：扩展（Extension）用于添加全新命令/工作流；预设（Preset）用于定制现有模板/格式；捆绑包（Bundle）则将扩展、预设、步骤等打包成角色化配置。
- **社区生态**：有官方文档站点，社区可贡献扩展、预设、捆绑包和演练指南。

**解决的问题**：
- 减少从需求到代码的认知鸿沟，让AI代理更准确理解意图。
- 避免“随性编码”（Vibe coding）导致的质量不可控。
- 标准化团队开发流程，提升协作效率。

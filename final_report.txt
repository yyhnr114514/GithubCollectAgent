   star: 14088  语言/技术: Python, PyPy3, MIT License
   项目简介: pytest 是一个让 Python 测试变得简单又强大的框架，既能轻松编写小型测试，也能支撑复杂的功能测试。它解决了传统 unittest 写法繁琐、断言信息不友好的痛点，是 Python 开发者的测试神器。
   亮点: 详细的断言失败信息（无需记忆 self.assert* 方法名，直接用 assert）; 自动发现测试模块和函数; 模块化 fixtures 管理测试资源; 兼容 unittest 测试套件; 支持 Python 3.10+ 和 PyPy3; 丰富的插件架构，拥有 1300+ 外部插件
   详细报告: pytest 是 Python 生态中最流行的测试框架之一，它的核心理念是让测试代码简洁易读。传统 unittest 需要继承 TestCase 类并使用 self.assert* 系列方法，而 pytest 允许开发者直接使用 Python 原生的 assert 语句，配合其强大的断言内省（assertion introspection）机制，失败时会自动展示详细的变量值和表达式求值过程，极大提升了调试效率。

项目支持自动发现测试模块和函数，无需手动配置测试套件。通过 fixtures 机制，开发者可以模块化地管理测试资源（如数据库连接、临时文件等），支持参数化和作用域控制。pytest 兼容 unittest 和 nose 测试套件，迁移成本低。

技术栈方面，pytest 基于 Python 3.10+ 或 PyPy3 运行，采用 MIT 开源协议。其插件架构极为丰富，拥有超过 1300 个外部插件，覆盖覆盖率报告（pytest-cov）、并行测试（pytest-xdist）、参数化测试（pytest-parametrize）等场景。社区活跃，文档完善，支持 Discord 和 Libera chat 社区交流。

   亮点: 覆盖 64 种漏洞模式、16 个安全类别，包括提示注入、数据外泄、权限提升、供应链攻击、MCP 工具投毒等 AI Agent 特有的安全风险; 双阶段检测管线：快速静态分析（高召回率）+ 可选 LLM 语义分析（精度提升至约 87%），兼顾效率与准确性; 支持多种输入格式（Git 仓库、URL、zip、目录、单文件）和多种输出格式（终端、JSON、Markdown、SARIF），可无缝集成 CI/CD 流程; 内置 SC4 实时漏洞查询，通过 OSV.dev API 检查依赖项的已知 CVE，支持离线回退; 提供 Python API（LangGraph 工作流），可作为库集成到自动化安全审计系统中
   详细报告: SkillSpector 解决的是一个随着 AI Agent 生态爆发而日益紧迫的安全问题：Agent Skill 以隐式信任方式执行，缺乏审查，而研究表明 26.1% 的 Skill 存在漏洞、5.2% 具有恶意意图。

**核心工作原理**：项目采用两阶段检测管线。第一阶段是静态分析，通过 11 个静态分析器进行正则匹配和 AST 行为分析，快速扫描所有文件，检测 exec/eval/subprocess 等危险调用、污点追踪、YARA 恶意软件签名匹配等，具有高召回率。第二阶段是可选的 LLM 语义分析，通过 OpenAI/Anthropic/NVIDIA 等提供商的模型评估上下文和意图，过滤误报，生成人类可读的解释，将精度提升至约 87%。LLM 提示中内置了反越狱防护，防止恶意 Skill 操纵分析过程。

**漏洞检测能力**：覆盖 16 个类别共 64 种模式——提示注入（5种）、数据外泄（4种）、权限提升（3种）、供应链（6种）、过度代理（4种）、输出处理（3种）、系统提示泄露（3种）、记忆投毒（3种）、工具滥用（3种）、流氓代理（2种）、触发器滥用（3种）、AST 行为分析（8种）、污点追踪（5种）、YARA 签名（4种）、MCP 最小权限（4种）、MCP 工具投毒（4种）。

**风险评分系统**：CRITICAL +50、HIGH +25、MEDIUM +10、LOW +5，可执行脚本乘以 1.3 倍系数，最终映射到 0-100 分数和四个严重级别（LOW/MEDIUM/HIGH/CRITICAL），给出明确的安装建议。

**工程化支持**：提供 Docker 镜像（基于 Python 3.12-slim-bookworm）、Makefile 构建流程、完整的 CLI 工具、Python API（LangGraph 工作流集成）、多格式报告输出（终端/JSON/Markdown/SARIF），支持本地 Ollama/vLLM 等 OpenAI 兼容端点。

**研究背书**：基于 Liu et al. 2026 的实证研究，分析了 42,447 个 Skill 的数据集，发现含可执行脚本的 Skill 存在漏洞的概率是普通 Skill 的 2.12 倍。

**局限性**：不支持非英文内容分析、无法分析图片中的文本、无法处理加密/二进制代码、纯静态分析不执行代码、离线时 SC4 漏洞查询受限于静态回退列表。

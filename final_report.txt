   详细报告: Headroom 的核心定位是'AI Agent 的上下文压缩中间件'。它解决的核心痛点是：当 AI Agent 处理复杂任务（如代码搜索、SRE 排障、GitHub issue 分类）时，会积累大量上下文（工具输出、日志、文件内容等），导致 token 成本剧增甚至超出模型上下文窗口。Headroom 通过在数据进入 LLM 前进行智能压缩，实测可减少 60%–95% 的 token 使用量，同时保持回答准确性（在 GSM8K、TruthfulQA 等基准上无损）。

技术实现上，Headroom 采用分层架构：ContentRouter 检测内容类型并选择最优压缩器；SmartCrusher 处理 JSON 结构数据；CodeCompressor 基于 AST 压缩代码；Kompress-base 是专为此场景训练的 HuggingFace 文本压缩模型；CacheAligner 优化 KV 缓存命中率；CCR（Contextual Compression with Retrieval）机制保证压缩可逆，原始数据本地存储，LLM 可通过 headroom_retrieve 工具按需获取。

部署方式极其灵活：可作为 Python/TypeScript 库直接调用 compress() 函数；可作为零代码修改的代理（headroom proxy）；可一键包装现有 Agent（headroom wrap claude）；也可作为 MCP 服务器集成到任何 MCP 客户端。支持跨 Agent 共享记忆（如 Claude 和 Codex 共享上下文），并具备 headroom learn 功能，能从失败会话中学习并自动更新 CLAUDE.md 等指令文件。

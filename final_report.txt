   项目简介: Headroom 是一个专为 AI Agent 设计的上下文压缩层，能在不影响回答质量的前提下，将发送给大模型（LLM）的 Token 数量削减 60%–95%。它通过本地优先的架构，对工具输出、日志、RAG 分块、文件及对话历史进行智能压缩，并提供可逆压缩（CCR）机制确保原始数据随时可取回，是降低 AI 应用运行成本、提升响应速度的“省钱神器”。
   亮点: 极致降本：实测最高可节省 92% 的 Token 消耗（如代码搜索场景），且基准测试显示准确率几乎无损（如 GSM8K 数学基准零损耗）。; 多模态压缩：内置 SmartCrusher (JSON)、CodeCompressor (代码AST) 和 Kompress-base (文本模型)，针对不同数据类型采用最优压缩算法。; 可逆压缩 (CCR)：原始数据永不丢失，LLM 可通过工具按需取回原文，兼顾了压缩率与数据完整性。; 无缝集成：支持 Proxy 模式（零代码改动）、Library 模式（Python/TS SDK）、Agent Wrap 模式（一行命令包裹 Claude/Cursor 等）以及 MCP Server。; 跨 Agent 记忆：支持 Claude、Codex、Gemini 等多个 Agent 共享上下文存储并自动去重。
   详细报告: Headroom 解决的核心痛点是：随着 AI Agent 处理的任务越来越复杂（如长对话、大量工具调用、代码库分析），发送给 LLM 的上下文（Context）会急剧膨胀，导致 API 成本飙升和响应延迟增加。

项目的技术壁垒在于其精细化的流水线设计：
1. **智能路由 (ContentRouter)**：自动识别输入内容是 JSON、代码还是普通文本。
2. **专用压缩器**：
   - **SmartCrusher**：针对 JSON 结构进行深度压缩，保留关键键值对。
   - **CodeCompressor**：基于 AST（抽象语法树）分析代码，移除注释、空白和不影响逻辑的冗余结构，支持 Python、JS、Go 等多种语言。
   - **Kompress-base**：自研的 HuggingFace 模型，专门用于压缩自然语言文本和 Agent 交互轨迹。
3. **缓存对齐 (CacheAligner)**：这是一个非常巧妙的工程优化。由于 LLM 提供商（如 Anthropic/OpenAI）通常有 KV Cache 机制，Headroom 会稳定前缀内容，确保压缩后的 Prompt 依然能命中 Provider 的缓存，进一步降低计费 Token。
4. **可逆压缩 (CCR)**：虽然压缩了，但原始文件存储在本地。如果 LLM 觉得信息不够，可以调用 `headroom_retrieve` 工具取回原文，实现了“该省省，该花花”。

此外，Headroom 提供了极其友好的接入方式。你可以把它当作一个本地代理（Proxy），完全不用改业务代码；也可以直接调用它的 Python/TS 库；甚至可以用 `headroom wrap claude` 一行命令直接给 Claude Code 套上“省钱 Buff”。它还具备 `headroom learn` 功能，能从失败的会话中提取教训并写入配置文件，让 Agent 越用越聪明。

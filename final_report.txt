   项目简介: Spec Kit 是一个由 GitHub 推出的开源工具包，旨在实现规范驱动开发（Spec-Driven Development），让开发者通过编写自然语言规范，由 AI 编码助手自动生成实现代码，从而减少“盲目编码”（vibe coding），提升软件质量。它提供 CLI 工具 specify 和一系列 slash 命令，与 30+ 种 AI 编码代理（如 Claude Code、GitHub Copilot）集成，支持扩展、预设和角色绑定包，方便团队定制开发流程。
   亮点: 规范即代码：将规范变为可执行，直接驱动 AI 生成符合需求的实现。; 零摩擦集成：与 30+ AI 编码代理深度集成，提供统一的 slash 命令工作流。; 灵活扩展：通过扩展（新增命令/模板）和预设（定制现有模板）满足不同团队需求，支持角色绑定包一键部署。
   详细报告: Spec Kit 是一个革命性的开发工具包，它重新定义了软件开发中规范与代码的关系。传统开发中，规范往往是“写后即弃”的文档，而 Spec Kit 将规范提升为一等公民，使其直接成为可执行的驱动源。

核心工具是 specify CLI，基于 Python 实现，通过 uv 工具安装。项目初始化后，开发者通过与 AI 编码代理交互使用一系列 slash 命令：
- `/speckit.constitution` 创建项目治理原则和开发指南；
- `/speckit.specify` 描述需求（仅关注“做什么”和“为什么”，不涉及技术栈）；
- `/speckit.plan` 指定技术栈和架构选择，生成实现计划；
- `/speckit.tasks` 将计划拆解为具体的任务列表，包含依赖关系和并行执行标记；
- `/speckit.implement` 按任务顺序自动执行编码。

这些命令会生成对应的 Markdown 文件（如 constitution.md、spec.md、plan.md、tasks.md），AI 代理根据这些文件逐步生成代码。支持 TDD 模式，先写测试再实现。

Spec Kit 的扩展系统允许社区贡献新命令、钩子和功能；预设系统可覆盖默认模板，适用于合规性、术语定制等场景；角色绑定包（Bundles）将扩展和预设打包，一键配置产品经理、分析师、开发者等不同角色的工具链。

技术方面，CLI 使用 Python，配置采用 YAML（如 bundle.yml），规范文件使用 Markdown。支持的 AI 代理包括 Claude Code、GitHub Copilot、Codex CLI 等 30 多种，通过插件机制集成。

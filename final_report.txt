   详细报告: 该项目名为 code-review-graph，旨在解决 AI 代码审查工具在审查时因重新读取大量代码库而浪费 token 的问题。其核心原理是利用 Tree-sitter 解析代码库生成抽象语法树（AST），并存储为包含节点（函数、类、导入等）和边（调用、继承、测试覆盖等）的结构化图。在每次代码变更时，通过该图进行最小上下文计算，只向 AI 助手提供影响范围内的文件，避免全量扫描。

主要功能包括：
- **增量更新**：通过 SHA-256 哈希检测变化文件，仅重新解析变更部分，2900 文件的仓库可在 2 秒内完成索引。
- **爆炸半径分析**：追踪变更影响的调用者、依赖项和测试文件，生成最小影响集。
- **语义搜索**：可选集成向量嵌入（如 sentence-transformers）提升搜索准确性。
- **多平台支持**：可配置用于 Codex、Cursor、Claude Code、GitHub Copilot 等主流 AI 编码工具。
- **GitHub Action**：在 PR 时自动生成风险评分、影响函数和测试缺口，可设置 fail-on-risk 作为合并门禁。
- **自定义语言支持**：用户可通过 languages.toml 添加新语言解析规则，无需修改源码。
- **基准测试**：在 6 个真实开源仓库上测试，平均 token 减少 82 倍，平均影响精度 F1 为 0.71。

技术栈主要基于 Python，依赖 Tree-sitter 解析器和 MCP 协议。安装简单（pip install code-review-graph），并使用 code-review-graph install 命令自动配置。项目文档详尽，包含 FAQ、使用指南、故障排除等，社区活跃度高（Discord）。

   项目简介: KTransformers 是一个面向大语言模型的高效推理与微调框架，核心是基于 CPU-GPU 异构计算，让超大规模 MoE 模型（如 DeepSeek-R1/V3）在消费级显卡（如 24GB VRAM）上也能跑得飞起，微调速度比 ZeRO-Offload 快 6-12 倍，简直是“打工人神器”。项目提供推理（kt-kernel）和微调（集成 LLaMA-Factory）两大能力，支持 Intel AMX/AVX 加速、MoE 优化、多级量化等，已集成到 SGLang 生产环境。
   亮点: CPU-GPU 异构推理，让超大 MoE 模型在低显存硬件上高效运行; 微调速度远超 ZeRO-Offload，支持 DeepSeek-V3/R1 等模型在 4×RTX 4090 上以 3.7 it/s 训练; 无缝集成 LLaMA-Factory 和 SGLang，降低生产部署门槛
   详细报告: KTransformers 是一个专注于大语言模型高效推理与微调的开源研究项目，核心创新在于利用 CPU-GPU 异构计算，将热门专家（hot experts）放在 GPU 上，冷门专家放在 CPU 甚至磁盘上，从而突破显存瓶颈。项目分为两大模块：

1. **推理 (kt-kernel)**：提供高性能异构推理服务。关键技术包括 Intel AMX/AVX 优化内核，支持 INT4/INT8 量化推理；针对 MoE 模型的 NUMA 感知内存管理；CPU-GPU 混合专家调度；支持多种精度（FP8、BF16、INT4/INT8）和量化格式（GPTQ、IQ1_S 等）。性能示例：DeepSeek-R1-0528 (FP8) 在 8×L20 GPU + Xeon Gold 6454S 上达到 227.85 tokens/s 总吞吐。集成到 SGLang 生产框架，并支持多并发、长上下文（24GB VRAM 下 139K tokens）。

2. **微调 (SFT)**：与 LLaMA-Factory 深度整合，实现超大规模 MoE 模型的 CPU-GPU 混合微调。支持 INT8/INT4 量化，显存占用极低——Qwen3-30B-A3B 可在单张 RTX 4090 上以 8+ it/s 微调，DeepSeek-V3 也仅需约 80GB 显存（4×RTX 4090）。通过 FSDP2 和 KTransformers 自定义优化器，训练速度比传统 ZeRO-Offload 快 6-12 倍，CPU 内存需求减半。

此外，项目持续跟进最新模型（如 MiniMax-M3、GLM-5、Kimi-K2 系列），并支持 AMD GPU（ROCm）、Intel Arc、Ascend NPU 等硬件。文档与社区活跃，有详细的教程和论文引用。因文档被截断，部分技术细节（如具体量化 kernel 实现）无法展开。

   项目简介: Cognee 是一个开源 AI 记忆平台，为 AI Agent 提供持久化长期记忆。它可以将任意格式的数据转化为自托管的知识图谱，让 Agent 跨会话记住、关联和行动。核心 API 包括 remember、recall、forget、improve，支持向量嵌入、图推理和本体生成。一句话：给 AI Agent 装上永不丢失的记忆。
   亮点: 统一记忆层：将关系、嵌入、会话、元数据全部运行在单一 Postgres 实例上，无需多数据库堆栈; 跨会话持久记忆：Agent 重启后仍能回忆起之前的对话和知识，适用于长期交互场景; 多语言客户端：官方支持 Python、Rust、TypeScript，并可作为 Claude Code 插件使用
   详细报告: Cognee 是一个开源的 AI 记忆平台，专为 AI Agent 设计，旨在解决长期记忆问题。它允许开发者将任意格式的数据（文本、文档等）注入系统，自动构建自托管的知识图谱，并结合向量嵌入和图推理，使文档既可通过语义搜索，又可通过关系连接。核心 API 非常简洁：`remember` 用于存储数据并构建知识图谱，`recall` 用于智能检索（自动选择最佳搜索策略），`forget` 用于删除记忆，`improve` 用于基于反馈优化记忆。

技术实现上，Cognee 默认使用 Postgres 作为统一后端，通过 pgvector 处理向量嵌入，通过内置图引擎管理关系，并通过 SQLite 缓存会话，从而简化部署。同时也支持 Neo4j、LanceDB、Redis 等专用组件。它支持多种 LLM 提供商（如 OpenAI），并通过环境变量配置。

部署方式灵活：可通过 pip 安装本地运行，也可使用 Docker 或一键部署到 Modal、Railway、Fly.io 等平台。此外，还提供 Cognee Cloud 托管服务。Cognee 还提供了针对 Claude Code 的插件，使 Claude 具备持久记忆能力。

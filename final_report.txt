   项目简介: LMCache 是一个专为大语言模型（LLM）推理设计的 KV 缓存管理层。它将原本易失的 KV 缓存转变为可持久化存储、跨引擎复用、可监控的“AI原生知识”，通过分层存储（GPU/CPU/SSD/远程）和智能复用机制，显著降低首字延迟（TTFT）并提升吞吐量，尤其适用于长上下文、多轮对话和 RAG 等场景。
   亮点: 引擎无关的独立守护进程架构：KV 缓存与推理引擎解耦，即使引擎崩溃缓存也不会丢失（无命运共享）; 持久化分层 KV 缓存卸载与复用：支持将 KV 缓存从 GPU 内存卸载到 CPU、本地磁盘、Redis、S3 等多级存储，实现跨请求、跨会话、跨引擎实例的复用，大幅减少重复预填充计算; 生产级 KV 缓存可观测性：提供丰富的监控指标，包括请求级/令牌级前缀缓存命中率、生命周期、用户用量等，便于运维诊断; 非前缀 KV 复用（CacheBlend）：突破传统前缀缓存限制，支持复用提示中任意位置的 KV 块，并通过选择性重计算保证生成质量; PD 分离与 KV 传输：支持通过 NVLink、RDMA 或 TCP 在预填充与解码工作节点间传输 KV 缓存，提升分布式推理效率; 插件化存储与传输后端：统一接口支持多种远程存储和传输后端，用户可自由切换供应商而不丢失缓存; 插件式 KV 变换（SERDE）：提供灵活的序列化/反序列化接口，支持研究人员实现压缩、令牌丢弃等自定义逻辑
   详细报告: LMCache 是一个专为大规模 LLM 推理优化的 KV 缓存管理中间件，解决了传统推理系统中 KV 缓存‘用完即弃’的痛点。其核心思想是将 KV 缓存视为可持久化、可共享、可管理的‘AI原生知识’，而非临时计算状态。

项目采用独立守护进程架构，与 vLLM、NVIDIA Dynamo 等主流推理引擎解耦，确保即使推理引擎崩溃，KV 缓存依然安全存在（无命运共享问题）。它支持将 KV 缓存从昂贵的 GPU 内存卸载到 CPU RAM、本地 SSD、Redis/Valkey、Mooncake、S3 对象存储等低成本层级，形成多级存储体系，从而在长上下文、多轮对话、RAG 等场景中实现跨请求、跨会话甚至跨引擎的缓存复用，显著降低首字延迟（TTFT）和计算开销。

LMCache 的一大创新是‘非前缀 KV 复用’（基于 CacheBlend 技术），突破了传统前缀缓存只能复用提示开头部分的限制，允许复用任意位置的 KV 块，并通过选择性重计算来恢复生成质量。此外，它支持 PD（Prefill-Decode）分离架构下的 KV 传输，可通过 NVLink、RDMA 或 TCP 在预填充与解码节点间高效传递缓存。

在可观测性方面，LMCache 提供了生产级的监控指标，涵盖缓存命中率、生命周期、用户级用量等，便于运维团队进行性能诊断和资源规划。其插件化设计允许用户轻松集成新的存储后端（如 Redis、S3）或传输层（如 NIXL），并通过 SERDE 接口实现自定义的 KV 变换（如压缩、令牌丢弃）。

LMCache 已加入 PyTorch Foundation，并被 NVIDIA Dynamo、CoreWeave、Cohere 等主流厂商集成，支持 AMD、Arm、Ascend 等多硬件平台，正成为 LLM 推理生态中事实上的 KV 缓存管理标准。

   项目简介: Agent Governance Toolkit (AGT) 是微软开源的 AI Agent 生产级治理中间件，解决自主 AI Agent 部署后的三大核心问题：权限控制（Agent 能做什么）、身份追溯（谁做的）、审计合规（能否证明发生了什么）。它通过在应用层拦截所有工具调用、消息发送和委托操作，以确定性策略引擎替代不可靠的 prompt 级安全，实现'让 Agent 不可能违规'而非'请求 Agent 守规矩'。支持 Python、TypeScript、.NET、Rust、Go 多语言，兼容主流 Agent 框架，提供策略执行、身份认证、沙箱隔离、SRE 监控、合规审计等全栈治理能力。

   详细报告: Agent Governance Toolkit 是微软为解决 AI Agent 生产部署中的治理难题而开发的开源中间件套件。其核心理念是：prompt 层的安全措施（如'请遵守规则'）对随机性的大语言模型而言只是礼貌请求，而非有效控制。AGT 将治理逻辑下沉到应用中间件层，在 Agent 的意图到达执行层之前进行拦截和判定。

**核心架构包含八大模块：**

1. **Agent OS（策略引擎）**：核心治理内核，支持 YAML/OPA/Cedar 策略定义，提供 fail-closed 语义（默认拒绝）。策略规则支持条件匹配、优先级排序、规则合并，并提供 68 个一致性测试。开发者只需两行代码 `govern(my_tool, policy="policy.yaml")` 即可为任意函数添加治理。

2. **Agent Mesh（身份与信任网格）**：基于 SPIFFE/DID/mTLS 实现零信任身份体系，解决多 Agent 共享 API 密钥时的归因难题。支持 Agent 发现、路由、信任评分和委托链追踪，提供 135 个一致性测试。

3. **Agent Runtime（执行沙箱）**：四级特权环（Privilege Rings）隔离机制，支持 Saga 编排、终止控制和执行计划验证。确保 Agent 在受限环境中执行，防止权限提升。

4. **Agent SRE（站点可靠性工程）**：为 Agent 提供 SLO 监控、错误预算、熔断器和混沌工程测试。包含紧急 Kill Switch，可在检测到异常时立即终止 Agent 行为。

5. **Agent Compliance（合规验证）**：自动化 OWASP 合规检查、策略文件完整性验证、策略 linting。内置 PromptDefense Evaluator，提供 12 维提示注入审计能力。

6. **Agent Marketplace（插件治理）**：插件生命周期管理和信任评分，防止恶意插件注入。

7. **Agent Lightning（RL 训练治理）**：为强化学习训练过程提供治理约束，对违规行为施加惩罚。

8. **Agent Hypervisor（执行审计）**：执行计划验证、可逆性验证和承诺锚定，确保操作可追溯、可回滚。

**关键差异化能力：**
- **MCP 安全网关**：检测工具投毒、漂移监控、域名抢注和隐藏指令扫描
- **Shadow AI 发现**：跨进程、配置和仓库扫描未注册 Agent

   详细报告: Agent Reach 本质上是一个 AI Agent 的互联网能力层（capability layer），它本身不负责底层内容读取，而是负责选型、安装、体检和路由。项目解决的核心痛点是：虽然 AI Agent 已经能写代码、改文档、管项目，但让它去网上找内容时就会抓瞎——Twitter API 要付费、Reddit 匿名接口被封、B站有风控、小红书要登录、网页抓回来一堆 HTML 标签没法读。每个平台都有自己的门槛，开发者需要逐个踩坑配置。

Agent Reach 把这些复杂性封装起来，提供了一键安装体验。用户只需把安装链接发给 Agent，它就会自动完成：安装 CLI 工具（pip install agent-reach）、安装系统基建（Node.js、gh CLI、mcporter）、配置搜索引擎（Exa 通过 MCP 接入，免费免 Key）、检测环境、注册 SKILL.md 使用指南。安装后 Agent 遇到全网调研、搜推特、看视频等需求时自动知道该调什么工具。

架构设计上，每个平台都维护一个首选+备选的有序后端列表。例如 B站从 yt-dlp 切换到 bili-cli（因为 yt-dlp 被 B站风控 412 封死），Twitter 首选 twitter-cli 备选 OpenCLI，小红书桌面首选 OpenCLI 服务器用 xiaohongshu-mcp。切换接入方式只需调整列表顺序，不需要重写代码。agent-reach doctor 会真实探测各候选后端可用性，第一个完整可用的当选，坏掉的给出修复处方。

支持的平台覆盖了主流内容平台：网页（Jina Reader）、YouTube（字幕提取+搜索）、RSS、全网语义搜索（Exa）、GitHub、Twitter/X、B站、Reddit、小红书、LinkedIn、V2EX、雪球、小宇宙播客等。零配置即可用的有 6 个渠道，需要登录的（Twitter、小红书、Reddit）通过 Cookie-Editor 导出 Cookie 配置。
